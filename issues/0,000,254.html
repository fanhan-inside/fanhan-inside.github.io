顺便抄一段日记，备份本人刚刚收到的情报，只是继续拓展脑洞，还没开始动手实践：

上网乱转时发现两条与自己需求有关的情报：

  1. Microsoft Keyboard Layout Creator到了1.4版，据说可以定制奇怪的布局。与2007年写《关于输入法》系列时相比（当时在日语布局下死活编译不通过，只能用WinDDK手改），应该有进步了吧？
  2. HargBuzz引擎可以处理序列输入到Unicode字符的转换，在2007年我设计键盘布局的时候提到过已经停止的在网页上直接转换文本的「Google Transliterate API」项目比较类似。

好了，又是停滞许久的计划可以继续推进了。这次按照最近这些年的原则，能用脚本用脚本，能调用库与模块就调用（甚至包括Office之类大号软件提供的组件），实在没辙了再「启动IDE建立项目写代码并编译生成」。话说回来，当年若是MSKLC好用，也不至于非得用WinDDK改源码了。

当然，这可能是昨天收拾整理了一大批C++的书有关，正如刚才一堆「学习语言」舆论当中特意提到拉丁语梵语一样，肯定与摄像头后的观众朋友与窃听器后的听众朋友实时观察我这里一小部分藏书当中的封面有关。

其中「当年（2007）“设计”的键盘布局」以及其它脑洞，在《关于输入法》七篇合集、第八篇开头备份所在的《囧斋随笔・说明〔零〕》、以及《穷哔与弱哔》当中提到了。那时候墙内可以直连谷歌，而Google Transliterate API是谷歌自己放弃的，而Microsoft Keyboard Layout Creator当时就是不好使，于是才会用WinDDK自力更生艰苦奋斗自己动手丰衣足食。
